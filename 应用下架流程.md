应用下架流程

```shell
1. 删除域名的Nginx配置和SSL证书
服务器上
rm -rf /etc/nginx/conf.d/admin.xxx
rm -rf /etc/nginx/conf.d/api.xxx
rm -rf /etc/nginx/conf.d/h5.xxx
rm -rf /etc/nginx/conf.d/agent.xxx
rm -rf /etc/letsencrypt/live/admin.xxx
rm -rf /etc/letsencrypt/live/api.xxx
rm -rf /etc/letsencrypt/live/h5.xxx
rm -rf /etc/letsencrypt/live/agent.xxx

本地执行
proxy-env项目下，将该项目的配置文件复制到 .env
cp .xx-xx.env .env
sh remote-delete.sh


1. 删除域名解析

2. 删除域名

3. 删除所有App

4. 删除项目代码

# 将xx-xxx 替换为项目名字
project_name=xx-xxx
cd /hk/${project_name}/env
docker-compose down
cd ../../
rm -rf /hk/${project_name}
rm -rf /hk/.${project_name}.*
rm -rf /root/.laradock/${project_name}
```
